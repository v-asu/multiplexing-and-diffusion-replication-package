# Replication Package

This is the replication package for "Multiplexing in Networks and Diffusion" by Arun G. Chandrasekhar, Vasu Chaudhary, Benjamin Golub, and Matthew O. Jackson.

The repository root contains the code, input data, and generated outputs for the retained empirical figures and tables used in the paper and supplement. Treat the folder containing `run_all.R` as the project root.

## Quick Start

From the repository root, restore the pinned R environment:

```sh
Rscript -e 'install.packages("renv", repos = "https://cloud.r-project.org"); renv::restore()'
```

The Python packages used by the simulation plotting script are listed in `requirements.txt`:

```sh
python3 -m pip install -r requirements.txt
```

Then run the workflow:

```sh
Rscript run_all.R
```

The workflow creates any missing output folders, installs missing R packages from CRAN if they are not already available, rebuilds the processed data, and regenerates the retained tables and figures. Python is called from within `run_all.R` for the simulation plots. If needed, set `REPLICATION_PACKAGE_PYTHON` to the path of the Python interpreter to use:

```sh
REPLICATION_PACKAGE_PYTHON=/path/to/python Rscript run_all.R
```

The package was tested from a clean copy by deleting `data/processed/`, `figures/`, and `tables/`, then running `Rscript run_all.R`. A successful run regenerates 15 LaTeX table files in `tables/` and 23 figure files in `figures/`.

## Folder Structure

- `code/`: R and Python scripts for data preparation, analysis, figures, and tables.
- `data/raw/`: package-local input data used to construct analysis datasets.
- `data/processed/`: derived datasets used by later scripts and regenerated by the workflow.
- `figures/`: regenerated figure outputs.
- `tables/`: regenerated LaTeX table outputs.
- `run_all.R`: runs the replication scripts in sequence from the package root.
- `DATA_DICTIONARY.md`: file-level description of raw inputs.
- `renv.lock`: pinned R package environment.
- `requirements.txt`: Python package requirements for simulation plotting.

## Expected Outputs

The main generated outputs are:

- `tables/tab_main_*.tex`: main-text empirical tables.
- `tables/tab_si_*.tex`: supporting-information tables.
- `figures/fig_main_*.pdf`: main-text empirical figures.
- `figures/fig_si_*.pdf`: supporting-information figures.

The generated files are included for convenience, but they are reproducible from the raw inputs using `run_all.R`.

## Data

The raw data are organized by source:

- `data/raw/rct_villages/`: RCT village network matrices, household and village covariates, and caste covariate CSVs.
- `data/raw/mf_villages/`: microfinance village network matrices, jati covariates, a precomputed pairwise household-distance file derived from GPS coordinates, a vertex-to-household key with coordinates removed, and wave 1 and wave 2 gender inputs. The raw GPS coordinates are not included in this public package.
- `data/raw/sims/`: precomputed simulation arrays used by `code/09_plot_sims.py`.

The empirical data used in this package come from earlier studies. The microfinance village data build on the Karnataka village network data used in Banerjee et al. (2013) and Banerjee et al. (2024). The RCT village network data and diffusion outcomes build on Banerjee et al. (2019).

The simulation arrays in `data/raw/sims/` were generated separately on a computing cluster using the scripts in `code/simulations/`. These cluster-side scripts are included for provenance and are not called by `run_all.R`. The local replication workflow starts from the precomputed `.npy` arrays and uses `code/09_plot_sims.py` to regenerate the simulation figures.

The processed data are organized by analysis sample:

- `data/processed/rct_villages/`: processed RCT adjacency matrices, edge lists, jati/backbone layers, diffusion centrality inputs, and union-analysis datasets.
- `data/processed/mf_villages/`: processed MF adjacency matrices, edge lists, and jati layers.

## Code

Shared setup lives in:

- `code/00_packages.R`: package bootstrap and library loading.
- `code/00_functions.R`: shared helper functions.

The workflow begins with:

- `code/01_data_prep.R`: constructs processed MF and RCT network objects from raw inputs.
- `code/02_pca.R`: creates PCA/loadings outputs.
- `code/layer_correlation.R`: creates network-layer correlation outputs.
- `code/03_backbone_with_jati.R` and `code/03b_backbone_no_jati.R`: construct backbone layers.
- `code/04_diffusion_centrality.R`: constructs diffusion-centrality inputs.
- `code/05_main_regs.R`: runs the main regression tables and plots.

Additional scripts create robustness checks, mechanisms, descriptive tables, gender analyses, network statistics, and simulation figures:

- `code/06_mpex_vs_diffusion.R`
- `code/06b_mpex_vs_diffusion_continuous.R`
- `code/06c_mpex_vs_diffusion_all_layers.R`
- `code/hh_diffusion_centrality.R`
- `code/complex_path_length_build.R`
- `code/figure_village_mpex_vs_lb_union.R`
- `code/figure_village_mpex_vs_complex_path_union.R`
- `code/table_individual_mplex_on_union_measures.R`
- `code/network_layer_stats.R`
- `code/network_stats.R`
- `code/gender_MF_wave1.R`
- `code/07_gender_MF_wave2.R`
- `code/gender_stratified_mpex.R`
- `code/08_mpex_vs_degree.R`
- `code/09_plot_sims.py`

Cluster-side simulation-generation scripts are stored separately:

- `code/simulations/sims_simple.py`: generates simple-contagion simulation arrays by village.
- `code/simulations/sims_complex.py`: generates complex-contagion simulation arrays by village.

```sh
Rscript -e 'install.packages("renv", repos = "https://cloud.r-project.org"); renv::restore()'
```
